    JavaScript Weekly #579

    2022/3/4

    (opens new window)

    TypeScript 4.6 Released (And It Even Benefits Non-TS Users!) (opens new window) — The typed superset of JavaScript takes another step forward with the addition of the es2022 target, allowing code in constructors before super(), improved recursion depth checks, more syntax errors that can be detected in JavaScript files (even without using TS itself!), and more.

    (opens new window)

    The Easiest Way to Monitor Node: Automatic Instrumentation (opens new window) — Monitoring your Node.js apps can be hard. You need to understand what you need to monitor, instrument code, and then make sense of all the data. Automatic instrumentation makes that whole process easier and streamlined. Find out how you can do it.

    # IN BRIEF:

    • 🇺🇦 The popular Leaflet (opens new window) mapping library was created by a Ukrainian developer (opens new window) and the Leaflet team has a message for JavaScript developers (opens new window) about the current invasion and things you can do to help Ukraine. We, too, have donated and will continue to do so. GitHub has also shared their response to the war. (opens new window)

    • Another month, another VS Code release. (opens new window) This time we get improved automatic language detection, lazy variable evaluation when debugging JS (and JS only, for now), and improved syntax error reporting in JS files.

    • 📆 DevOps.js (opens new window) is a virtual conference on building, deploying and monitoring JS apps taking place in just three weeks (March 24-25). You can register here. (opens new window)

    • Apple, Bocoup, Google, Mozilla and others have announced Interop 2022 (opens new window), a collaboration to improve interoperability of (mostly visual, for now) Web technologies.

    • The much loved and heavily used MDN Web Docs (opens new window) site now has a new homepage, design, and logo.

    Reveal.js 4.3: An HTML Presentation Framework (opens new window) — Brings elegant presentations to anyone with a Web browser. v4.3 (opens new window) just came out with a few tweaks.

    Redux Toolkit 1.8.0 Released (opens new window) — If you’re down with Redux for managing state, the “official, opinionated, batteries-included toolset for efficient Redux development” is what you need 😃 Among other things, this release adds new ‘listener’ middleware (think useEffect but for Redux store updates).
