    2022/2/25

    😔 这对世界来说是艰难的一天，因此，我们与所有受欧洲目前正在发生的事件影响的人同在。 __
    Peter Cooper — 编辑

    Node v17.6.0 (Current) 发布 (opens new window) — 添加对 通过 HTTPS 导入 ES 模块的实验性支持 (opens new window) — Hemanth HM 在 Twitter 上有一段视频 (opens new window) 展示了此功能。 npm 升级到 8.5，并且像往常一样修复了许多小错误。

    Caxa：将 Node 应用程序打包成可执行二进制文件 (opens new window) — 如果您想创建 Node 应用程序的单个可执行版本，有很多选择，包括 Vercel 的 pkg (opens new window) 或 nexe (opens new window)。 Caxa 是一个固执己见的替代方案，它有一个非常详尽的 README，准确地涵盖了它的 工作原理 (opens new window) 以及 它与 pkg 和 nexe 的比较 (opens new window)。

    来自 RedisGreen 的安全 Redis 托管 (opens new window) — 通过最新的 Redis 功能保护 Redis 托管，使用一些工具进行检测和扩展。

    如何使用 npm 覆盖修复安全漏洞 (opens new window) — package.json 文件中的 Overrides (opens new window) 允许您将依赖关系树中的包替换为您选择的另一个版本，这可能是解决具有已知安全问题的依赖关系版本所必需的。

    Next.js 12.1 发布 (opens new window) — 流行的 React 框架引入了按需增量静态再生 (ISR) 的 beta 版本，现在可以自动为您配置 Jest，并且可以为您的应用创建更小、更高效的 “独立” 构建以用于生产部署。

    # 简介:

    • 在 Twitter 上，Simon Plenderleith 展示了一个使用 Node 的 新 AbortSignal API 取消异步操作 (opens new window)的便捷技巧。

    ▶  3.5 小时 Nest.js 初学者课程 (opens new window) — Nest.js (opens new window) 是一个流行的框架，用于使用 Node 构建后端应用程序和 API，这个视频将逐步介绍如何使用它构建您自己的 REST API，包括身份验证和数据库。

    如何在 AWS Lambda 上运行每个 Node.js 版本 (opens new window) — Everynode (opens new window) 提供了一种在 AWS Lambda 上使用 v11 及更高版本的任何 Node 版本的方法。

    使用 Group Step 类型更好地组织大型和复杂的管道 ✨ (opens new window)
